——————<<< V2ray 开启 TLS >>>————————-

cd /home/

git clone https://github.com/letsencrypt/letsencrypt

cd letsencrypt

./letsencrypt-auto certonly –standalone –email your@email.com -d www.yourdomain.com #若无法安装，试试停止iptables

证书目录通常为/etc/letsencrypt/live/www.yourdomain.com/ #证书只有3个月有效期 ！

vim config.json # 配置服务端。增加以下代码

“streamSettings”: {
“network”: “tcp”,
“security”: “tls”,
“tlsSettings”: {
“serverName”: “www.yourdomain.com",
“certificates”: [
{
“certificateFile”: “/etc/letsencrypt/live/www.yourdomain.com/fullchain.pem",
“keyFile”: “/etc/letsencrypt/live/www.yourdomain.com/privkey.pem"
}
]
}
},

v2ray 端口必须设为443

reboot #重启

============================heroku + snova 代理===========================================

首先安装 Java

下载必要工具

Snova代理： http://code.google.com/p/snova/downloads/list # snova-c4-java-server-0.22.0.war

windows snova客户端 gsnova_0.22.1_windows_386.zip

下载并安装Heroku Toolbelt工具：https://toolbelt.heroku.com/download/windows

安装完毕

开始-运行-cmd

heroku login

输入heroku 账号密码

email : **
password : **

安装heroku-deploy

heroku plugins:install https://github.com/heroku/heroku-deploy

或者

heroku plugins:install heroku-cli-deploy

创建应用

heroku apps:create

**-**-数字.herokuapp.com 是应用地址，记下来

上传服务端snova-c4-java-server-0.22.0.war ，假如下载后的服务端在D根目录，则

要挂VPN

d:

heroku deploy:war –war ./snova-c4-java-server-0.22.0.war –app --数字 #应用地址,后面无herokuapp.com

heroku deploy:war –war <path_to_war_file> –app <app_name>

例如 heroku deploy:war –war ./snova-c4-java-server-0.22.0.war –app serene-ocean-30485

完成后打开 serene-ocean-30485.herokuapp.com 若出现

snova c4 server 0.22.0
………
………

domainname ____

则部署成功 ！

解压 snova windows 客户端 gsnova_0.22.1_windows_386.zip

打开gsnova.conf

GAE Enable=1 的1改为 0

C4 Enable=0 的0改为 1

WorkerNode[0]= 此处填上应用地址 --数字.herokuapp.com

SPAC Enable=1 改为 0

127.0.0.1 默认端口 48102 # gsnova.conf里有标注

打开 gsnova.exe

成功
